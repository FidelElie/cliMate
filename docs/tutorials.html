Getting Started

Hello World With Some More Flavour

The minimal example on the homepage is the application without the explanation. So firstly, lets start by creating this application taking care with the steps. Lets start by making a python file. Name it whatever you like and open a terminal in your desired project space. Then enter the command:


clm new-app

After this you will get a confirmation message in your terminal saying that the cli.json file has been created in the target location:

CliMate App Created In Root

This is the heart of your new command line application and has its own syntaxes and structures for defining your commands. The anatomy of this file will be discussed throughout the tutorial series, but for now lets add our first command. I would suggest for the first few examples, declarations should be made manually so you can familiarise yourself with the structure of the cli file and writing it. Below shows our first command, copy this into the "commands" object in the cli.json file we created previously. 


"say-hello": {
    "name": "Say Hello",
    "description": "Say Hello To Someone Or Something",
    "target": {
        "type": "function"
    },
    "arguments": {
        "someone": {
            "name": "Someone",
            "description": "Someone's name or a generic word",
            "type": "str",
            "default": "World"
        }
    }
}

The command "say-hello" is now registered for your command line application. Now we can hook everything up in python. Open your python file that we created previously and add the snippet below:


from climate import CliMate

def say_hello (name):
    print("Hello, {}!".format(name))

if __name__ == "__main__":
    climate = CliMate("cli.json")
    climate.parse_args()

Your command Line application is now ready to go so now you can run the command:

This command will print "Hello World" to your terminal but we also defined an extra argument for the name. So you can add any name you wish after the command and it will print it. This is the basics of how to set up your command line application but lets look more into it. Now we can also check the help function by typing.

Additionally, we can also access this command by opening the command line application's Main Menu. This is achieved by inputting no commands or arguments into the cmd.

This is the basics of how to start your own CliMate application but lets look at what we have written so far with close scrutiny.

A Closer Look

As you have seen already CliMate revolves around commands. Each of which holds a name, description, target and optional arguments. Instead of just passing your desired data directly to the file object everything is organised by these commands. The command call is defined by its key in the case of the example above this is say-hello. The "name" and "description" keys are used in creating the Menu and Help structures and should be brief but concise.

The target key is where the magic happens and deserves its own section entirely to see what it is capable of ["Add Scroll Link"]. However, for now its purpose is to tell the application where it should send the arguments too. This can be in a function as in the example above or other first class object like methods and classes. Above we define the target type as a function. Usually we can explicitly state the name of the function that we want to pass the arguments to. However, in this example this key being omitted means that CliMate will search for a function that is the same as your command with any hyphens being replaced with underscores (say-hello becomes say_hello).

Arguments are defined in a similar matter to the command itself. They possess a name and a description for the same purposes. As well as keys for defining the desired data type of the input and default values if desired.

Summarising

To summarise what we have gone through so far, the general structure of our cli file is:


{
    "general": {
        // general application information is defined here.
    },
    "command": {
        "command-1": {
            "name": "Command Name",
            "description": "Command Description",
            "target": {
                "type": "Target Type"
            },
            "arguments": {
                "argument-1": {
                    "name": "Argument Name",
                    "description": "Argument Description",
                    "type": "Argument Data Type"
                }
            }
        }
    }
}

All of this will become easier to utilise as you go through the tutorials on offer her. The example above is a good way to show the basics of what can be done but noe lets look more into creating commands.
